# Reshop 数据库表字段说明

本文档对应 SQL 文件：[reshop_schema.sql](D:\gostudy\Re_Shop\script\reshop_schema.sql)

## 1. `users` 用户表

用于保存商城用户的基础信息，并通过简单角色字段区分普通用户和管理员。

| 字段名 | 中文意思 | 说明 |
| --- | --- | --- |
| `id` | 用户ID | 用户表主键，自增唯一标识。 |
| `username` | 用户名 | 用户登录账号，要求唯一。 |
| `password` | 密码 | 用户登录密码，实际项目里应保存加密后的密码。 |
| `nickname` | 昵称 | 用户在系统中展示的名称。 |
| `avatar` | 用户头像 | 用户头像图片地址，可保存图片 URL 或文件路径。 |
| `phone` | 手机号 | 用户绑定手机号，要求唯一。 |
| `role` | 用户角色 | 用户权限角色，当前简单设计中 `0` 表示普通用户，`1` 表示管理员。 |
| `status` | 用户状态 | 用户当前状态，通常 `1` 表示正常，`0` 表示禁用。 |
| `created_at` | 创建时间 | 用户记录首次创建的时间。 |
| `updated_at` | 更新时间 | 用户记录最近一次更新的时间。 |

## 2. `categories` 商品分类表

用于保存商品分类信息，支持一级分类和多级分类。

| 字段名 | 中文意思 | 说明 |
| --- | --- | --- |
| `id` | 分类ID | 分类表主键，自增唯一标识。 |
| `name` | 分类名称 | 商品分类的名称，例如“手机”“家电”。 |
| `parent_id` | 父分类ID | 上级分类ID，`0` 一般表示顶级分类。 |
| `sort` | 排序值 | 控制分类展示顺序，值越小通常越靠前。 |
| `created_at` | 创建时间 | 分类记录创建时间。 |
| `updated_at` | 更新时间 | 分类记录最后更新时间。 |

## 3. `products` 商品表

用于保存商品的核心信息。

| 字段名 | 中文意思 | 说明 |
| --- | --- | --- |
| `id` | 商品ID | 商品表主键，自增唯一标识。 |
| `name` | 商品名称 | 商品正式名称。 |
| `title` | 商品标题 | 商品副标题或简短卖点说明。 |
| `category_id` | 分类ID | 商品所属分类的ID。 |
| `price` | 销售价 | 商品当前销售价格。 |
| `original_price` | 原价 | 商品原始价格，用于显示优惠前价格。 |
| `cover_image` | 商品封面图 | 商品主图地址或图片链接。 |
| `detail` | 商品详情 | 商品详细描述，可保存图文说明文本。 |
| `status` | 商品状态 | 商品当前状态，通常 `1` 表示上架，`0` 表示下架。 |
| `created_at` | 创建时间 | 商品记录创建时间。 |
| `updated_at` | 更新时间 | 商品记录最后更新时间。 |

## 4. `product_stock` 商品库存表

用于单独管理普通商品库存，便于后续扩展扣减和锁定逻辑。

| 字段名 | 中文意思 | 说明 |
| --- | --- | --- |
| `id` | 库存记录ID | 库存表主键，自增唯一标识。 |
| `product_id` | 商品ID | 对应的商品ID，和商品一一对应。 |
| `stock` | 可用库存 | 当前可销售的库存数量。 |
| `locked_stock` | 锁定库存 | 已被下单但尚未最终完成支付的库存数量。 |
| `created_at` | 创建时间 | 库存记录创建时间。 |
| `updated_at` | 更新时间 | 库存记录最后更新时间。 |

## 6. `orders` 订单表

用于保存订单主信息。

| 字段名 | 中文意思 | 说明 |
| --- | --- | --- |
| `id` | 订单ID | 订单表主键，自增唯一标识。 |
| `order_no` | 订单编号 | 对外展示和业务查询使用的唯一订单号。 |

## 7. `order_items` 订单明细表

用于保存订单中的每一项商品明细。

| 字段名 | 中文意思 | 说明 |
| --- | --- | --- |
| `id` | 明细ID | 订单明细表主键，自增唯一标识。 |
| `order_id` | 订单ID | 所属订单的ID。 |

| `product_price` | 商品单价 | 下单时商品的成交单价。 |
| `quantity` | 购买数量 | 该商品购买了多少件。 |
| `subtotal_amount` | 小计金额 | 该商品项的小计金额，通常等于单价乘数量。 |

| 字段名 | 中文意思 | 说明 |

## 补充说明

### 状态字段建议

- `users.role`：`0` 普通用户，`1` 管理员
- `users.status`：`1` 正常，`0` 禁用
- `products.status`：`1` 上架，`0` 下架
- `orders.status`：`0` 待支付，`1` 已支付，`2` 已取消，`3` 已完成
- `seckill_activities.status`：`0` 未开始，`1` 进行中，`2` 已结束

### 设计思路说明

- `users.role` 先采用简单字段设计，便于当前阶段快速做管理员和普通用户区分，后续需要更复杂权限时再升级为 RBAC。
- `product_stock` 单独拆表，是为了后面普通购买和秒杀扣库存更清楚。
- `order_items` 冗余保存商品名称和价格，是为了保证历史订单数据稳定。
- `seckill_orders` 主要用于防止同一用户重复抢购同一个秒杀商品。
